[Seu Nome] - Documentação de Sandbox Web Analytics

Demonstração de implementação de Tags GTM e rastreamento de eventos personalizados no GA4.

1. Visão Geral: Criação de um Ambiente de Teste (Sandbox)

Este repositório no GitHub Pages serve como um ambiente controlado para simular o comportamento de um site real e testar a injeção e o disparo de Tags sem impactar a produção.

Passos Fundamentais (O Que Aprendemos):

  • Escolha da Plataforma: Usar o GitHub Pages para garantir acesso irrestrito ao código HTML (incluindo o <head>).
  • Geração da URL: Obter uma URL pública (https://) para que o GA4 e o Tag Assistant aceitem a conexão (resolvendo o problema do file:///).
  • Implementação: Inserir o snippet do GTM diretamente no <head> para garantir a inicialização correta e a detecção pelo Tag Assistant.

2. O Código: Injeção e Camada de Dados (DataLayer)

Os testes de Tags dependem do dataLayer para enviar informações do site para o GTM.

Função do Botão (Exemplo de Implementação):

Ao clicar nos botões abaixo, um evento personalizado é enviado ao dataLayer, que o GTM capta para disparar a Tag do GA4.

A estrutura de um evento personalizado é: dataLayer.push({'event': 'nome_do_evento', 'parametro': 'valor'}).
